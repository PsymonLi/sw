Workload

The "Workload" refers to the virtual machine that will be managed by Venice. The workload is defined by the following attributes.

Option Description
name Name of the object, unique within a Namespace for scoped objects. Should start and end in an alphanumeric character and can contain alphanumeric or .- characters. minimum length is 2 and maximum length is 64 charactersName of the object, unique within a Namespace for scoped objects. Should start and end in an alphanumeric character and can contain alphanumeric or .- characters. minimum length is 2 and maximum length is 64 charactersmust start and end with alpha numeric and can have alphanumeric, -, _, .length of string should be between 2 and 64
host-name Hostname of the server where the workload is running. cli-help: Host name where the workload runsHostname of the server where the workload is running. cli-help: Host name where the workload runsshould be a valid host address, IP address or hostname

Interfaces

List of the interfaces for the workload. Click on "+Interface" to create a new interface for the workload

Option Description
mac-address MACAddress contains the MAC address of the interface as seen by the workload. cli-help: MAC address of the interface as seen by the workloadMACAddress contains the MAC address of the interface as seen by the workload. cli-help: MAC address of the interface as seen by the workloadshould be a valid MAC address
micro-seg-vlan Micro-segmentation vlan assigned for this interface. cli-help: Vlan identifying host unique vlan idMicro-segmentation vlan assigned for this interface. cli-help: Vlan identifying host unique vlan idvalue should be between 1 and 4095
external-vlan External vlan assigned for this interface. cli-help: External vlan associated with the workloadExternal vlan assigned for this interface. cli-help: External vlan associated with the workloadvalue should be between 0 and 4095
ip-addresses List of all IP addresses configured on a Workload InterfaceList of all IP addresses configured on a Workload Interface
  • Name: Unique name for the workload
  • Host Name: This will be a drop-down box listing the available hosts to associate the workload
  • Interfaces:
    • Mac-Address: mac-address of the workload (vm)
    • Micro-seg vlan: This will be an internal vlan id that will be unique to provide secure segmentation from other workloads (vms) within the same physical server.
    • External vlan: This vlan id is the vlan number correlating with the upstream switch or top of rack switch
    • IP Addresses: IP Address of the workload

Click on "Create Workload" to complete the task.

Examples:

Host Name: Prod-Server1

Interfaces:

Mac-Address: 0000.2a2d.9af1
Micro-seg vlan: 10
External vlan: 100
