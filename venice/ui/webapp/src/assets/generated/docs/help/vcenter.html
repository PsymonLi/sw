vCenter

The vCenter sub-section under the controller section refers to the VMware vCenter that we want to manage. Assumption is that the virtual infrastructure admin would have created a cluster of DSC hosts under a DataCenter, and PSM admin will register the vCenter as part of the VMware vCenter integration.

Add vCenter

vCenter is created with the following attributes:

Option Description
name Name of the object, unique within a Namespace for scoped objects. Must start and end with alpha numeric and can have alphanumeric, -, _, . Length of string should be between 2 and 64.
uri URI of the orchestrator. Length of string should be at least 1.
DataCenter(s) to Manage DataCenter(s) selection by either

"Manage All Datacenters" - To manage all DataCenters within the vCenter

"Manage Individual Datacenters" - To manage one or more DataCenter (space separation)
  • Credentials: Credential can be
Option Description
None No Credentials
Username/Password Username and Password of the vCenter
Token Token for authentication and authorization
Certs Certs comprise of Private key, Cert, and CA Bundle

vCenter Integration Overview displays the following five columns for all the registered vCenters.

Option Description
Name vCenter name
creation-time CreationTime is the creation time of the object. System generated and updated, not updatable by user.
uri URI of the orchestrator. Length of string should be at least 1.
connection-status
Valid options include:
- Failure
- Success
- Unknown
last-connected
