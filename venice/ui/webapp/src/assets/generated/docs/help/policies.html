Network Security Policies

In this Network Security Policy section, network security rules are created. The security policy rules can be associated to either a Security Group or a Tenant. To create the various rules to the security policy, they will have the following attributes.

Option Description
name Name of the object, unique within a Namespace for scoped objects. Must start and end with alpha numeric and can have alphanumeric, -, _, . Length of string should be between 2 and 64.
Rule Number The order of this rule. Must be an integer.
Action The rule action, etierh Permit, Deny, or Reject.
Deny - Packet will be dropped and no response is sent back to the source.
Reject - Packet will be dropped but a packet "port unreachable" will be sent to the source.
Permit - Allows the packet to be forwarded
apps List of apps objects to which the rule applies to.
proto-ports List of (protocol, ports) pairs to which the rule applies to, in addition to apps.
action SGRule action, either PERMIT, DENY or REJECT.
Valid options include:
- Deny
- Permit
- Reject
from-ip-addresses Inbound rule from a given ip-address/ip-mask/ip-range. Use any to refer to all ipaddresses cli-tags: id=from-ip.
to-ip-addresses Outbound rule from a given ip-address/ip-mask/ip-range. Use any to refer to all ipaddresses cli-tags: id=to-ip.
from-security-groups Inbound rule from a given security group.
to-security-groups Outbound rule from a given security group.

To add another rule, click on "+RULE".

Example:

Name: DC1-Security

Attach: Tenant

Rules:

1   
    Action: Permit
    From: 0.0.0.0/0
    To: DHCP-Servers
    Proto-Ports:
        Protocol: udp
        Ports: 67-68
2
    Action: Permit
    From: 10.29.100.0/24
    To: 10.29.50.101 10.29.50.102
    Proto-Ports:
        Protocol: tcp
        Ports: 80, 443
